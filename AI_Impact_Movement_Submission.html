Key Code — CDD/MDD Delivery Count Filter

The most complex filter rule — two condition groups with blank-field fallback — translated directly from plain-language description to working code:

Python · pandas
base_filter = (
    df["Task Type"].isin(["SOW_GF", "SOW_BF"]) &
    df["Task Status"].isin(["Skeleton HO Completed", "System HO Completed"])
)

# Group A: MDD falls within the reporting week
group_a = df[base_filter & df["MDD"].between(week_start, week_end)]

# Group B: MDD is blank — fall back to CDD
group_b = df[base_filter & df["MDD"].isna() & df["CDD"].between(week_start, week_end)]

delivery_count = len(group_a) + len(group_b)
Lessons Learned
  • Describing business logic in plain language is enough — you don't need to know how to code to build a working automation. The AI handles the translation.
  • Breaking the problem into small steps (filter first, then classify, then output) made it easier to verify each part before moving on — a useful habit for any AI-assisted development.

Automating Weekend Support Filtering: From Manual Excel Work to 2-Minute Script Execution

Team: ECS TSM · Basis Intern · Helen Fu

2
Problem Statement

As a Basis intern in the TSM team, I support the Weekend Safeguard Support process every week. One recurring task is Weekend Support Filtering — extracting and processing scheduled maintenance tickets for the upcoming weekend from a SharePoint-hosted Excel file.

The manual workflow involves:
  • Copying raw ticket data from SharePoint into a local Excel template, then manually fixing structural issues introduced by the paste: removing merged cells, deleting a duplicate header row, and removing a duplicate column — every single week.
  • Applying multi-step filtering: removing blacklisted customers, isolating tickets within the Saturday/Sunday 00:00–09:00 time window, classifying each ticket by customer type (Critical / Normal) and task type (Standard / Non-Standard).
  • Manually color-coding each row based on classification, applying a grouping-based deletion rule, sorting results by Customer Name, and generating the final output file.
Pain Points
  • ~15 minutes per run, every week, 52 times per year
  • The grouping-based deletion rule is easy to apply incorrectly when done manually
  • Template structural issues must be fixed from scratch every single time
  • Pure data transportation and formatting — no value-added work
3
Current State / Proposed Solution
Current State
  • Open SharePoint file, copy all ticket rows into local template manually
  • Fix merged cells, duplicate header row, and duplicate column by hand each time
  • Filter rows by date and time window (00:00–09:00 Saturday and Sunday)
  • Classify each row by reading Critical_customer and standard work reference sheets
  • Color-code each row individually based on classification
  • Apply grouping rule for row deletion, checking each customer group manually
  • Sort by Customer Name, save structured output with separate sheets per day
Proposed Solution — 1_Automation tool.py
Preprocessing: Automatically unmerges all merged cells, detects and removes the duplicate header row, removes duplicate columns — zero manual template fixes required.
Filtering & Classification: Reads blacklist from PC3&Hybrid sheet, filters by time window (00:00–09:00 Sat/Sun), classifies tickets using Critical_customer and standard work reference sheets.

Key Code — Grouping-based Row Removal

The most non-trivial business rule, translated directly from plain-language description to working code:

Python · 1_Automation tool.py
def remove_normal_standard_rows(df):
    def is_normal_standard(row):
        return row['Customer_Type'] == 'Normal' and row['Service_Type'] == 'Standard'

    # Per customer: True only if ALL rows are Normal + Standard
    customer_all_ns = df.groupby('Customer Name').apply(
        lambda g: g.apply(is_normal_standard, axis=1).all(),
        include_groups=False
    )

    # Delete a row only if it IS Normal+Standard AND its entire customer group is too
    mask_delete = df.apply(
        lambda row: is_normal_standard(row) and customer_all_ns.get(row['Customer Name'], True),
        axis=1
    )
    return df[~mask_delete].reset_index(drop=True)
Lessons Learned
  • File must be closed in Excel before the script saves (OS file lock) — correctly identified by Claude Code from the error message
  • pandas 2.2+ requires include_groups=False in groupby().apply() — version-specific fix surfaced and resolved immediately during testing
  • Trailing whitespace in source data silently splits one customer into two groups, causing incorrect row deletions — fixed by normalizing with .strip() before groupby
